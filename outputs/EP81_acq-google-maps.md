## Background

> 2003: PM Bret Taylor's "Search by Location" had zero users. Yahoo Maps push and Sequoia intro to Where 2 — Lars and Jens Rasmussen's dynamic tile map in Noel Gordon's Sydney bedroom. Larry Page meeting: AJAX rewrite satisfied "we like the web." Parallel buys: Keyhole (Earth Viewer, Hanke), ZipDash (4 Nextel phones traffic). Feb 2005 Maps beta Slashdotted; satellite view drove "find my house" viral adoption. 2006: largest online mapping provider; Maps API unleashed mashup era (PadMapper, Trulia). Dependency on Tele Atlas/NavTeq ended via Street View cars, Ground Truth (Megan Quinn), VuTool — booting vendors Oct 2009. Mobile: iPhone 2007 shipped Maps with Google data; Android 2009 added turn-by-turn ( withheld from iOS during platform war). Apple Maps 2012 disaster → Google native iOS app 10M installs in 2 days. Waze acquired 2013 ($1.3B).

## Key Facts

> **F1** Three 2004 acquisitions: Where 2 (undisclosed, estimated <$50M), Keyhole ≈$35M, ZipDash ≈$2M — ZipDash became dominant mobile Maps despite seeming insignificant.

> **F2** Google Maps API launched mid-2006; enabled mashups (PadMapper, Zillow, Trulia) — foundational to Uber/DoorDash-class businesses per episode.

> **F3** October 2009: Google replaced Tele Atlas/NavTeq with proprietary map data from Street View / Ground Truth programs.

> **F4** December 2012: Google Maps iOS app reached 10M+ installs within ≈2 days of Apple booting Google from default Maps on iOS 6.

> **F5** Current API pricing cited ≈$7 per 1,000 requests; ≈5M customer API keys — local ads and API dual revenue vs years of running Maps deeply unprofitable.

## Mental Model · *Maps as Platform, Not Feature*

> **Components**
>
> Lars insight: big interactive map is the UI platform, not printable directions. AJAX tiles + Web 2.0 made 100× UX jump. Google bundled acquisitions under Taylor/Hanke geo org, then invested billions in owned data (cars, satellites) to escape vendor tax. Maps becomes second search box (local intent) plus B2B API layer capturing ≈10% of value created (Shopify 2.5% analogy). Moat = data + latency + distribution (Android, default iOS era then win-back app). Competitors need capital and time Google had as profitable search subsidized bet.
>
> **Application**
>
> Platform acquisitions should be evaluated on ecosystem enablement, not standalone P&L early. Owning core data beats licensing when suppliers can choke strategic products. In mobile OS wars, controlling maps + navigation is strategic weapon (Apple learned painfully).

## What Makes It Work

> Google Maps stacks proprietary global basemap, real-time traffic (ZipDash/Waze), Street View imagery, routing scale, and developer API standardization. Engineering depth (vector tiles post-static era) and Search integration for local ads. Weaknesses: multi-billion CapEx; regulatory scrutiny on bundling; Apple continues improving Maps; Mapbox well-funded but uphill. Self-driving tailwind slower than expected when episode recorded. Versus Instagram A+: Maps is high fixed-cost infrastructure with lower margin profile than pure software social — but wider enabling impact across economy. Skybox Imaging (2014) satellites; Ground Truth derived map geometry from Street View stills. Lars → Google Wave (Slack precursor) → Facebook Workplace → weav.io running app. Foursquare 10.0 rating carve-out request for Google PMs. Waze $1.3B vs ≈$10–20M Where 2 — acquisition price irrelevant versus platform value. Self-driving tailwind slower than 2012 hype when episode recorded.

## Key Insights

> **1.** Sequoia passed Where 2 but won anyway via Google intro.
>
> **Q** Could Where 2 have stayed independent?
>
> **A** Pre-2004 capital markets couldn't fund global map data buildout; Sequoia skeptical of map business models. Flickr/YouTube sold early too — Maps needed Google's loss tolerance and search cash flow.

> **2.** ZipDash looked trivial, became mobile Maps core.
>
> **Q** Why pay attention to tiny acquisitions?
>
> **A** $2M ZipDash (4 Nextel phones) foreshadowed mobile dominance — 90%+ usage on phones today. Integration into backwater mobile team inverted significance vs web launch.

> **3.** Apple Maps proved data moat length.
>
> **Q** Could Apple displace Google quickly?
>
> **A** iOS 6 (2012) lacked decade of Google Ground Truth, user probes, and routing ML — turn-by-turn sent users wrong places. Google shipped full iOS app in 3 months using same API backend; 10M installs in 2 days.

> **4.** Maps API created more value than Google captured early.
>
> **Q** Value creation vs capture?
>
> **A** Free then cheap API spawned trillion-dollar adjacent markets (Uber, etc.); Google ran Maps deeply red ≈10+ years. Capture rising via ads + $7/1K API — platform captures small slice of enormous enabled GMV.

> **5.** Alumni tree spans Salesforce to Niantic.
>
> **Q** Who came from Maps?
>
> **A** Bret Taylor → FriendFeed → Facebook CTO → Quip → Salesforce CPO; John Hanke → Niantic/Pokémon Go; Megan Quinn → Square/KPCB; Lars → Wave → Facebook Workplace → weav.io. Geo group as Google talent factory.
